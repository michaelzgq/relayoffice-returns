## One-Line Offer

`We review how your warehouse handles returned items after they arrive, then show you exactly where grading, evidence capture, refund release, and SLA control are breaking down.`

中文：

`我们不替换你现有的退货系统，只帮你看清退货到仓后这段判断流程哪里在漏证据、漏规则、漏 SLA、漏钱。`

## Who To Sell First

### Best ICP

| 维度 | 标准 |
|---|---|
| 公司类型 | 中小 3PL / returns-focused 3PL |
| 客户结构 | 服务 `5-30` 个 DTC 品牌 |
| 当前状态 | shopper-side returns 可能已有，但仓库侧 grading / disposition / evidence 仍靠 SOP / sheet / notes |
| 高信号 | 每个品牌规则不同、refund hold 不清楚、fraud cases 证据分散、SLA backlog 不透明 |
| 对接角色 | 3PL owner、warehouse ops lead、returns manager、client success lead |

### Do Not Start With

| 对象 | 原因 |
|---|---|
| 单品牌 DTC 商家 | 多规则复杂度不够高 |
| enterprise reverse logistics 团队 | 销售周期过长 |
| 只想优化 shopper portal 的品牌 | 不是你的战场 |

## What You Are Actually Selling

这不是软件 demo。  
这是一份有明确输出物的 review。

### Review Includes

| 交付物 | 说明 |
|---|---|
| `Workflow Gap Map` | 梳理退货到仓后的现有流程，标出规则断点和责任断点 |
| `Evidence Checklist` | 定义 empty box / wrong item / missing parts / damage 的标准证据要求 |
| `Refund Gate Recommendation` | 哪些 case 应在 inspection 前 hold，哪些可自动放 |
| `Brand Rules Risk Map` | 哪些品牌规则依赖口头培训、sheet 或个人记忆 |
| `SLA Risk Snapshot` | 哪些阶段容易超时、积压、无人跟进 |
| `Chargeback-Ready Sample` | 给出一个标准化证据包样例 |
| `30-minute Review Call` | 解释问题和下一步建议 |

### What It Does Not Include

- shopper portal redesign
- label workflow
- exchange logic
- full software rollout
- full chargeback filing for every processor

## Review Format

### Recommended Structure

1. `Current State`
当前他们怎么做 returns inspection / grading / refund release

2. `Observed Risks`
当前最大的规则、证据、SLA、fraud 风险

3. `Priority Fixes`
先修哪 `3` 件事最值钱

4. `Sample Evidence Pack`
展示一份标准化 case 长什么样

5. `Next-Step Recommendation`
是否值得进入 setup / monthly support / tool pilot

## Review Checklist

### Discovery Inputs To Request

| 输入 | 作用 |
|---|---|
| 现有 returns SOP / playbook | 看规则是不是碎 |
| `1-3` 个品牌的 returns rules | 看规则差异 |
| `3-5` 个真实 returns case | 看现场证据和判断方式 |
| refund release 逻辑 | 看谁决定什么时候放钱 |
| SLA 目标 | 看有没有 backlog 风险 |

### Questions To Ask On The Call

1. 现在每个品牌的 grading 规则是谁维护？
2. 仓库人员怎么知道一件退货该 restock、hold、还是 destroy？
3. empty box / wrong item / missing parts 的证据现在放在哪里？
4. refund 是 inspection 后决定，还是先自动放？
5. 你们最常被品牌 challenge 的 returns case 是哪类？
6. 目前有没有哪种 case 经常超过 SLA？
7. 如果新品牌明天上线，你们要花多久把 returns 规则教给仓库？

不要卖：

- “我们有个 returns 软件”

要卖：

- `We help 3PLs standardize what happens after returned items hit the warehouse.`
- `Most teams have a shopper-facing portal, but the warehouse-side decision layer is still held together with SOPs, sheets, and tribal knowledge.`

```text
We’re looking at one very specific problem for multi-brand 3PLs:
what happens after a returned item reaches the warehouse.

Not labels or customer-facing returns.
The warehouse-side rules:
grading, evidence capture, refund hold/release, and SLA risk.

I’m offering a small paid Returns Decision Support Review to show where those rules are leaking time, money, or evidence.

If you’re open, I can walk through your current process and return a concise review with the top gaps and a sample chargeback-ready evidence pack.
```

### Even Shorter Version

```text
Quick question: once returned items reach your warehouse, how are grading rules, evidence capture, and refund holds managed across different brands today?

That’s the exact workflow I’m reviewing for multi-brand 3PLs right now.
```

## Two-Week Plan

### Week 1

1. 拉 `20` 家目标 3PL 名单
2. 每家只找 `1-2` 个最相关的人
3. 发出第一轮触达
4. 约 `5` 个 discovery calls

### Week 2

1. 跑完 `5` 个 call
2. 推出 `Pilot Review $300`
3. 目标至少拿到：
   - `2` 个愿意看 sample
   - `1` 个真实付费 review

## Pass / Kill

### Pass

| 条件 | 标准 |
|---|---|
| 有效 discovery calls | `>= 5` |
| 愿意看样例 | `>= 2` |
| 付费试单 | `>= 1` |

### Kill

| 条件 | 结论 |
|---|---|
| `2 周内` 没有 `5` 个有效 call | `NO-GO` |
| 大多数人说仓库侧 rules 已很清楚 | `NO-GO` |
| 大多数人更关心 shopper portal 而不是 warehouse-side decisioning | `NO-GO` |
| 没人愿意为 review 付费 | `NO-GO` |
