# Dossentry Narrow ICP Reprioritization - 2026-04

Last updated: 2026-04-18 America/Los_Angeles

## Goal

Re-rank the current outbound list around the narrower ICP that is most likely to buy Dossentry early:

**small-to-mid multi-brand 3PLs with explicit returns handling, client-facing exception pressure, and no heavyweight reverse-logistics stack**

This file is not a new market map.
It is an execution filter for who to contact first.

## Narrow ICP

An account should move up only if it matches most of these:

1. `3PL / fulfillment operator`, not software vendor or brand-only team
2. Explicit `returns management / reverse logistics / return processing / inspection`
3. Clear signal that warehouse staff inspect, grade, photo, re-kit, or decide next step
4. Multi-brand or client-facing operating model
5. Likely still running on SOPs, email, folders, and team memory rather than a heavyweight reverse-logistics suite
6. Buyer access is realistic for founder-led outbound

## Tier Logic

### `A` Tier

Contact now.
Best match for the current wedge:

- warehouse-side exception control
- evidence capture
- brand/client explanation
- phone-first inspection

### `B` Tier

Keep in active rotation.
Real fit, but weaker on one of these:

- buyer accessibility
- explicit client-facing exception pressure
- likely company size / complexity
- directness of the evidence wedge

### `C` Tier

Do not prioritize in the first wave.
Still relevant, but more likely to dilute message testing.

## Exact First-Wave ICP

If you want one sentence to screen future accounts, use this:

> `U.S. multi-brand 3PLs that already process returns in-house, visibly inspect or document exception cases, and still need a cleaner way to explain those cases back to brand clients.`

## Who To Exclude For Now

Do not spend early outreach time on:

- enterprise-scale reverse logistics providers
- single-brand operators
- companies that only emphasize shopper returns portals
- freight / brokerage / transport-only operators
- operators whose public story is mainly system integration, not warehouse exception handling
