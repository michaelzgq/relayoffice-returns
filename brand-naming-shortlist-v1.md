# Brand Naming Shortlist V1

## Goal

在 `7` 天内选出一个比 `RelayOffice` 风险更低、可用于公开推广和产品上线的新品牌方向。

这份文档的目标不是“直接定最终名字”，而是：

1. 停止继续押 `RelayOffice`
2. 给出低风险命名规则
3. 给出可筛选的候选清单
4. 选出最值得继续做 deeper clearance 的 `5` 个名字

## Why RelayOffice Is A No-Go

- `Relay` 在物流、returns、通讯等相关商业语义带里已经明显拥挤。
- `relaytech.co` 已在做 delivery / returns 相关服务。
- `relayoffice.net` 已被实际使用。
- USPTO 官方明确建议 clearance 不能只看注册，还要看互联网使用、域名、common-law use。

这意味着：

- 法律不确定性高
- 客户认知混淆高
- SEO 和口碑传播效率低

结论：

**`RelayOffice` / `RelayOffice Returns` 不适合作为正式对外品牌继续推进。**

## Naming Rules

### Must-Have

- 不包含 `Relay`
- 不把 `Returns` 当品牌主体
- 可以作为 umbrella brand 扩到别的 workflow
- 听起来像 B2B 软件，不像消费者电商品牌
- 适合搭配副标题：
  - `Brand-ready return evidence and decision workflows`
  - `Decision-ready case records for return disputes`

### Avoid

- `Relay + anything`
- `Returns + anything` 作为主品牌
- 太像 WMS / shipping / carrier 的名字
- 太像 AI 聊天工具或消费者 App 的名字
- 全通用词堆叠，比如 `ProofLink`、`CaseGrid`、`RuleSystem`

### Preferred Style

- `2-3` 音节
- 稍微抽象，但仍然能联想到审计、证据、规则、判断
- 更像“系统名”而不是“功能名”

## Reject Now

这些名字或方向不建议再继续投入：

| Name / Pattern | Why Reject |
|---|---|
| `RelayOffice` | 与 `Relay` 相关现有使用和行业语义冲突过高 |
| `RelayOffice Returns` | 同时踩中 `Relay` 和 `Returns` 两个拥挤词 |
| `Relay + logistics/returns word` | 在当前赛道中过于拥挤，解释成本高 |
| `Veridock` | 容易落到物流/港口/仓位语义，且命名拥挤 |
| `Docketry` | 已有互联网使用痕迹，不适合当首选 |
| `RuleGrid` | 过于通用，也有明显互联网痕迹 |
| `Evidentra` | 已出现互联网痕迹，不适合继续押 |
| `Casevera` | 已出现互联网痕迹，不适合继续押 |
| `Fieldmark` | 已存在明显商用名称痕迹 |
| `Averiq` | 已出现相似痕迹，不适合作为低风险首选 |

## Shortlist

这 `5` 个是我认为最值得继续做 deeper clearance 的方向。

### 1. Northset

**为什么排第一**

- 不被 `returns`、`logistics`、`proof` 这些拥挤词绑死
- 更像可扩展的公司或产品 umbrella brand
- 适合 premium、克制、B2B 的感觉

**适合的对外表达**

- `Northset`
- `Northset for return evidence and decision workflows`

**风险**

- 语义没有直接说明产品，需要副标题帮助理解

### 2. Rulenta

**为什么排第二**

- 保留了你现在产品最核心的 playbook / rules 优势
- 听感比 `RuleGrid`、`RuleSystem` 更像品牌而不是功能名
- 适合企业软件

**适合的对外表达**

- `Rulenta`
- `Rulenta helps 3PLs standardize disputed return handling`

**风险**

- 仍然带一点“rule”语义，需要避免做得太像纯 rules engine

### 3. Proofora

**为什么排第三**

- 很接近你现在最值钱的资产：evidence、proof、shareable case record
- 对品牌 review link 的定位很贴合

**适合的对外表达**

- `Proofora`
- `Proofora turns warehouse return cases into brand-ready evidence`

**风险**

- `proof` 是强语义词，商标独特性天然比纯抽象词弱

### 4. Caselith

**为什么排第四**

- 听感偏稳重、偏系统化，不像营销词
- 适合你这种 case-centric 产品

**适合的对外表达**

- `Caselith`
- `Caselith gives ops teams a defensible case record`

**风险**

- `case` 也是泛词，独特性中等

### 5. Gradevia

**为什么排第五**

- 很贴合 grading / recommendation / decision-support 方向
- 对你当前的 warehouse inspection 语义比较顺

**适合的对外表达**

- `Gradevia`
- `Gradevia helps teams document and review high-risk returns`

**风险**

- 如果未来你不想被“grading”框住，这个名字扩展性会弱于 `Northset`

## Recommendation

### Primary

**`Northset`**

最像可以一路走下去的 umbrella brand，也最不容易被当前品类困住。

### Backup

**`Rulenta`**

如果你希望品牌和产品优势更贴近，`Rulenta` 是更强的业务贴合型名字。

### Evidence-Led Option

**`Proofora`**

如果你决定把 `Brand Review Link` 和 evidence layer 放到绝对核心，这个名字会更有销售力。

## 48-Hour Next Steps

1. 从 shortlist 里只保留 `3` 个：
   - `Northset`
   - `Rulenta`
   - `Proofora`
2. 对这 `3` 个做：
   - 精确 Google / LinkedIn 搜索
   - 域名可用性检查
   - USPTO 检索
3. 不要在产品里继续使用 `RelayOffice`
4. 在律师正式 clearance 之前，代码里如果需要临时占位，先用中性 placeholder，不要再公开推一个可能有风险的名字

## Temporary Placeholder Advice

在正式品牌确定前，建议临时对外使用：

- `Brand Review Demo`
- `Decision Control Demo`
- `Return Evidence Demo`

不要继续把 demo 顶在 `RelayOffice Returns` 上扩大传播。

## Final Verdict

**现在最对的动作不是继续推广 `RelayOffice`，而是立刻切到一个不含 `Relay` 的新品牌方向。**

如果只保留一个起跑方向：

**先查 `Northset`。**
